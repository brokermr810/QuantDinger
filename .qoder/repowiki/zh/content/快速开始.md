## 简介
QuantDinger 是一个自托管、本地优先的量化交易操作系统：集 AI 辅助研究、Python 原生策略、回测与实盘执行（加密、IBKR 股票、MT5 外汇）于一体的一体化 Compose 栈。本文档提供“从零到运行”的快速开始指南，帮助你在 2 分钟内完成安装、配置与首次验证，并给出生产部署前置知识与注意事项。

## 系统要求与先决条件
- Docker 与 Docker Compose v2（Windows/macOS 使用 Docker Desktop，Linux 使用 Docker Engine + Compose 插件）
- Git
- 不需要 Node.js（默认已内置预构建前端）

以下步骤适用于 macOS/Linux 与 Windows（PowerShell/Git Bash）两种方式。你也可以直接使用根 README 提供的一行命令。

### 步骤概览
1) 克隆仓库并进入目录  
2) 复制并编辑后端环境模板为 .env  
3) 生成并写入 SECRET_KEY（必须修改为非默认值）

5) 首次登录并修改默认管理员密码

## 常见问题排查
- 后端启动即退出
  - 可能原因：SECRET_KEY 仍为默认值，或 .env 语法错误
  - 排查方法：查看后端容器日志；确认 .env 已写入非默认 SECRET_KEY
- 浏览器打开白屏或出现 API 错误
  - 可能原因：FRONTEND_URL 与实际访问域名不一致，或 API 无法从浏览器访问
  - 排查方法：核对 backend_api_python/.env 中 FRONTEND_URL；确认端口映射与防火墙
- 端口被占用
  - 可能原因：本地已有服务占用了 5432/6379/5000/8888
  - 排查方法：在根目录 .env 中调整对应端口映射（参考 docker-compose.yml 的端口定义）
- 并发策略过多导致“启动被拒”
  - 可能原因：同时运行的策略过多
  - 排查方法：提高 backend_api_python/.env 中的 STRATEGY_MAX_THREADS 并重启后端

## 生产环境部署要点
- 网络与端口
  - 建议将前端/后端/数据库绑定到 127.0.0.1，仅对外暴露 80/443，由宿主机 Nginx 反代
  - 参考根目录 .env 的端口映射示例，避免直接将内部端口暴露到公网
- HTTPS 与反向代理
  - 使用宿主机 Nginx + Let’s Encrypt 获取免费证书
  - 将 HTTPS 终止于宿主机 Nginx，容器内仅监听 127.0.0.1
- 镜像拉取加速
  - 若 Docker Hub 拉取缓慢，可在根目录 .env 设置 IMAGE_PREFIX 指向镜像源
- 日志与持久化
  - 数据库与后端日志目录已在 docker-compose.yml 中声明为命名卷，确保重启后数据不丢失
- 安全加固
  - 修改默认管理员密码
  - 严格控制 FRONTEND_URL 与 OAuth 回调域
  - 如启用 AI 功能，配置至少一个 LLM 提供商密钥
